FAQ

Frequently Asked Questions

Does the comparison need exact matches?
By default yes, but you can enable fuzzy matching to catch slight spelling variations or name formatting differences.
Is the comparison case-sensitive?
No. The tool compares values in a case-insensitive way by default, so 'JOHN SMITH' and 'John Smith' are treated as a match.
Can I compare by multiple columns (e.g., first name AND last name)?
Yes. You can concatenate columns or use a combined key column before running the check.
What does the output file contain?
Two tabs: 'Found' with all matched records, and 'Not Found' with all unmatched records — plus summary counts on a Run Summary tab.
